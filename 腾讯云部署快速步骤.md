# 🚀 腾讯云部署快速步骤

**预计时间**：15-20分钟

---

## 📋 快速开始（5步完成）

### ✅ 第1步：创建腾讯云COS存储桶（5分钟）

1. **访问COS控制台**
   - https://console.cloud.tencent.com/cos
   - 登录腾讯云账号

2. **创建存储桶**
   - 点击"创建存储桶"
   - **名称**：`yqf-aibtrip-demo`（或自定义，需全局唯一）
   - **地域**：`广州`（或离您最近的地域）
   - **访问权限**：**公有读私有写**（重要！）
   - 点击"创建"

3. **开启静态网站托管**
   - 进入存储桶 → "基础配置" → "静态网站"
   - 开启静态网站托管
   - **索引文档**：`index.html`
   - **错误文档**：`index.html`（SPA需要）
   - 保存

---

### ✅ 第2步：获取API密钥（2分钟）

1. **访问API密钥管理**
   - https://console.cloud.tencent.com/cam/capi

2. **创建密钥**
   - 点击"新建密钥"
   - 复制并保存：
     - `SecretId`
     - `SecretKey`

**⚠️ 重要：请妥善保管密钥，不要泄露！**

---

### ✅ 第3步：配置GitHub Secrets（3分钟）

1. **访问GitHub仓库设置**
   - https://github.com/HMC1234/-yqf-aibtrip-demo/settings/secrets/actions

2. **添加Secrets**
   - 点击"New repository secret"
   
   **添加以下4个Secrets**：
   
   | Name | Value |
   |------|-------|
   | `TENCENT_SECRET_ID` | 您的SecretId |
   | `TENCENT_SECRET_KEY` | 您的SecretKey |
   | `TENCENT_BUCKET` | 您的存储桶名称（例如：`yqf-aibtrip-demo`） |
   | `TENCENT_REGION` | 您的地域（例如：`ap-guangzhou`） |

---

### ✅ 第4步：提交工作流文件（2分钟）

工作流文件已创建在`.github/workflows/deploy-tencent.yml`

**提交并推送**：
```bash
git add .github/workflows/deploy-tencent.yml
git commit -m "feat: 添加腾讯云COS自动部署工作流"
git push origin main
```

---

### ✅ 第5步：等待自动部署（3-5分钟）

1. **查看部署状态**
   - 访问：https://github.com/HMC1234/-yqf-aibtrip-demo/actions
   - 查看最新的工作流运行

2. **部署成功后访问**
   - COS默认域名：
    ```
    https://<您的存储桶名称>.cos-website.<您的地域>.myqcloud.com
    ```
   - 示例：
    ```
    https://yqf-aibtrip-demo.cos-website.ap-guangzhou.myqcloud.com
    ```

---

## 🎯 完成！

部署完成后，每次推送到`main`分支都会自动部署到腾讯云！

---

## 📝 下一步（可选）

### 配置CDN加速（推荐）

1. **访问CDN控制台**
   - https://console.cloud.tencent.com/cdn

2. **添加域名**
   - 加速域名：例如 `demo.yqf-aibtrip.com`
   - 源站类型：COS源
   - 选择您的存储桶

3. **配置HTTPS**
   - 开启HTTPS（可使用腾讯云免费证书）

4. **等待部署**（10-30分钟）

## ❓ 遇到问题？

### 问题1：部署失败 - 认证失败

**解决**：检查GitHub Secrets中的SecretId和SecretKey是否正确

### 问题2：网站无法访问

**解决**：
- 检查存储桶权限是否为"公有读私有写"
- 检查静态网站托管是否已开启

### 问题3：路由404错误

**解决**：确保在静态网站设置中，错误文档设置为`index.html`
