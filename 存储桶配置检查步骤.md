# 📋 存储桶配置检查步骤

**错误**：`NoSuchBucket` - 存储桶不存在

---

## 🔍 快速检查步骤

### 步骤1：查看您的存储桶信息（2分钟）

1. **访问COS控制台**
   ```
   https://console.cloud.tencent.com/cos
   ```

2. **查看存储桶列表**
   - 在存储桶列表中，找到您创建的存储桶
   - **记录存储桶名称**：例如 `my-bucket-name`
   - **记录存储桶地域**：例如 `广州` 或 `北京`

3. **查看存储桶详情**
   - 点击存储桶名称进入详情页
   - 在"基础配置"中确认：
     - **存储桶名称**：例如 `yqf-aibtrip-demo` 或其他
     - **所属地域**：例如 `广州（ap-guangzhou）` 或 `北京（ap-beijing）`

---

### 步骤2：确认GitHub Secrets配置（2分钟）

1. **访问GitHub Secrets**
   ```
   https://github.com/HMC1234/-yqf-aibtrip-demo/settings/secrets/actions
   ```

2. **检查以下Secrets**

   **必需的Secrets**：
   - ✅ `TENCENT_SECRET_ID` - 应该已配置
   - ✅ `TENCENT_SECRET_KEY` - 应该已配置

   **可选的Secrets**（如果与默认值不同，需要配置）：
   - ⚠️ `TENCENT_BUCKET` - 如果存储桶名称不是 `yqf-aibtrip-demo`，需要配置
   - ⚠️ `TENCENT_REGION` - 如果地域不是 `ap-guangzhou`，需要配置

3. **如果存储桶名称或地域与默认值不同，添加Secrets**

   **添加 `TENCENT_BUCKET`**（如果存储桶名称不是 `yqf-aibtrip-demo`）：
   - 点击"New repository secret"
   - Name: `TENCENT_BUCKET`
   - Value: 您的实际存储桶名称（从步骤1获取）
   - 点击"Add secret"

   **添加 `TENCENT_REGION`**（如果地域不是 `ap-guangzhou`）：
   - 点击"New repository secret"
   - Name: `TENCENT_REGION`
   - Value: 您的地域代码（从步骤1获取，例如：`ap-beijing`）
   - 点击"Add secret"

---

### 步骤3：再次运行工作流（1分钟）

1. **访问GitHub Actions**
   ```
   https://github.com/HMC1234/-yqf-aibtrip-demo/actions
   ```

2. **运行工作流**
   - 点击"Deploy to Tencent Cloud COS"
   - 点击"Run workflow"
   - 选择分支：`main`
   - 点击"Run workflow"

3. **查看日志**
   - 等待部署完成
   - 查看日志中的"存储桶"和"地域"信息
   - 确认与您的实际配置一致
