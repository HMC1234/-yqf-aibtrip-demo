# 🔍 如何查看GitHub Actions工作流

**问题**：在GitHub Actions页面看不到"Deploy to Tencent Cloud COS"工作流

**原因**：工作流文件还没有推送到GitHub（因为GitHub推送保护）

---

## 📋 问题原因

工作流文件（`.github/workflows/deploy-tencent.yml`）在本地存在，但还没有成功推送到GitHub。

这是因为之前GitHub推送保护检测到了文档中的示例密钥模式，阻止了推送。

---

## ✅ 解决方案

### 方案1：解决GitHub推送保护问题（推荐）⭐

由于工作流文件本身没有问题，我们需要允许GitHub推送包含示例密钥的文档。

**步骤**：

1. **访问GitHub推送保护页面**
   ```
   https://github.com/HMC1234/-yqf-aibtrip-demo/security/secret-scanning/unblock-secret/35kZr13D1u5yIkArn2B6gr3nZKU
   ```

2. **允许密钥**
   - 点击"Allow this secret"按钮
   - 这些只是文档中的示例说明，不是真实密钥
   - 选择"Allow this secret"原因：These are example placeholders in documentation

3. **重新推送**
   - 允许后，再次推送代码
   - 或者我可以帮您推送

---

### 方案2：只推送工作流文件（快速方案）⭐

我可以帮您创建一个新的提交，只包含工作流文件，不包含其他可能有问题的文档。

**步骤**：

1. **只添加工作流文件**
   ```bash
   git add .github/workflows/
   git commit -m "feat: 添加腾讯云COS自动部署工作流"
   git push origin main
   ```

2. **检查是否成功**
   - 访问：https://github.com/HMC1234/-yqf-aibtrip-demo/actions
   - 应该能看到"Deploy to Tencent Cloud COS"工作流

---
