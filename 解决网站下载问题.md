# 🔧 解决网站下载问题

**问题**：访问网站地址时，浏览器下载文件而不是显示网页

**原因**：COS存储桶的**静态网站托管**功能没有开启

**解决方案**：在COS控制台开启静态网站托管功能

---

## ✅ 快速解决步骤（5分钟）

### 步骤1：访问COS控制台

**访问**：
```
https://console.cloud.tencent.com/cos
```

---

### 步骤2：进入存储桶配置

1. **点击存储桶**
   - 点击存储桶名称：`yqf-aibtrip-demo-1387623418`

2. **进入静态网站设置**
   - 左侧菜单：**基础配置**
   - 找到 **静态网站托管** 选项
   - 点击 **编辑** 或 **配置**

---

### 步骤3：开启静态网站托管

1. **开启功能**
   - 点击 **编辑** 按钮
   - **状态**：选择 **开启**
   - **索引文档**：输入 `index.html`
   - **错误文档**：输入 `index.html`（重要！SPA需要）
   - **强制HTTPS**：建议开启

2. **保存配置**
   - 点击 **保存** 按钮
   - 等待配置生效（通常立即生效）

## ❓ 常见问题

### Q1: 为什么文件被下载？

**A**: 因为静态网站托管功能未开启。当静态网站托管未开启时，COS会返回文件的Content-Type为 `application/octet-stream`（下载类型），浏览器就会下载文件而不是显示网页。

### Q2: 开启静态网站托管后多久生效？

**A**: 通常立即生效，但可能需要等待几秒钟。

### Q3: 为什么需要设置错误文档？

**A**: 对于SPA（单页应用），所有路由都会返回 `index.html`，这样React Router才能正确路由。如果不设置错误文档，访问 `/dashboard` 等路由时会出现404错误。

### Q4: 访问地址是什么？

**A**: 
```
https://<存储桶名称>.cos-website.<地域>.myqcloud.com
```

您的地址：
```
https://yqf-aibtrip-demo-1387623418.cos-website.ap-guangzhou.myqcloud.com
```
