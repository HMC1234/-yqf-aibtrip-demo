# 检查启动问题

## 如果页面打不开，请检查：

### 1. 检查项目是否正在运行

打开新的终端窗口，执行：
```bash
netstat -ano | findstr :3000
```

如果有输出，说明项目正在运行。

### 2. 检查编译错误

查看运行 `npm start` 的终端窗口，看是否有错误信息。

常见错误：
- **Module not found** - 缺少文件
- **Syntax error** - 语法错误
- **Type error** - 类型错误

### 3. 手动启动（如果自动启动失败）

停止当前进程（Ctrl+C），然后：

```bash
# 清除缓存
npm cache clean --force

# 删除node_modules（如果需要）
rmdir /s /q node_modules

# 重新安装
npm install --legacy-peer-deps

# 启动
npm start
```

### 4. 检查浏览器控制台

打开浏览器开发者工具（F12），查看：
- Console标签 - 是否有JavaScript错误
- Network标签 - 是否有请求失败

### 5. 尝试其他端口

如果3000端口被占用，可以修改端口：

在 `package.json` 中修改：
```json
"scripts": {
  "start": "set PORT=3001 && react-scripts start"
}
```

然后访问 http://localhost:3001

## 常见问题

### Q: 页面显示空白
**A**: 检查浏览器控制台，可能有JavaScript错误

### Q: 显示 "Cannot GET /"
**A**: 项目可能没有正确启动，检查终端错误

### Q: 显示编译错误
**A**: 查看终端中的具体错误信息，通常是：
- 缺少导入
- 语法错误
- 类型错误
