# 🔄 重启开发服务器说明

## ⚠️ 重要提示

由于我们刚刚添加了代理配置文件 (`src/setupProxy.js`)，**必须重启开发服务器**才能让代理配置生效。

## 📋 重启步骤

### 方法1: 手动重启（推荐）

1. **找到运行 `npm start` 的终端窗口**
   - 如果服务器正在运行，你会看到类似这样的输出：
     ```
     Compiled successfully!
     ```

2. **停止服务器**
   - 在终端窗口中按 `Ctrl + C`
   - 等待服务器完全停止

3. **重新启动服务器**
   ```bash
   npm start
   ```

4. **等待编译完成**
   - 看到 "Compiled successfully!" 消息
   - 浏览器会自动打开 http://localhost:3000

### 方法2: 如果找不到终端窗口

1. **打开新的终端窗口**
   - 在项目根目录下

2. **查找并停止进程**
   ```bash
   # Windows PowerShell
   Get-Process -Name node | Where-Object {$_.Path -like "*node.exe*"} | Stop-Process -Force
   ```

3. **重新启动**
   ```bash
   npm start
   ```

## ✅ 验证代理是否生效

重启后，打开浏览器开发者工具（F12），在Console中应该看到：
- 没有CORS相关错误
- 如果看到代理日志，说明代理已生效

## 🧪 测试步骤

1. **访问测试页面**
   ```
   http://localhost:3000/test/yqf-api
   ```

2. **填写配置**
   - API Base URL: `https://bizapi.yiqifei.cn/servings`
   - App Key: 你的App Key
   - App Secret: 你的App Secret
   - 点击"保存配置"

3. **测试航班查询**
   - 切换到"航班查询"标签
   - 填写查询参数
   - 点击"查询航班"

4. **检查结果**
   - 在Network标签中，请求URL应该是: `http://localhost:3000/api/yqf?...`
   - 不应该再出现CORS错误

---

**提示**: 如果重启后仍然有问题，请检查：
1. `src/setupProxy.js` 文件是否存在
2. 终端中是否有代理相关错误
3. 浏览器控制台的完整错误信息
